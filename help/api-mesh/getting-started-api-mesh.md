# Introduzione a mesh API

Se hai poca esperienza con API Mesh, Adobe consiglia di iniziare con questa esercitazione introduttiva prima di accedere ai relativi video e tutorial.

## Cosa è la mesh API

API Mesh consente l’utilizzo di più origini di dati per fornire una singola risposta da utilizzare per l’applicazione commerce.

[Visualizza la documentazione completa di API Mesh](https://developer.adobe.com/graphql-mesh-gateway/gateway/overview/)

## Esempi di casi d’uso

Il sistema back-end per la tua applicazione Commerce dispone di un’API REST da utilizzare per ottenere prezzi speciali. Un altro sistema di back-end con un endpoint GraphQL gestisce lo stato di inventario. Utilizzando la mesh API, puoi definire entrambi gli endpoint, recuperare le informazioni e restituirle all’applicazione richiedente come un’unica risposta.

## Cos&#39;è un proxy inverso

In qualità di sviluppatore che utilizza Adobe App Builder e GraphQL Mesh, non è necessario capire cosa sia un proxy inverso. Tuttavia, potresti essere interessato alla funzionalità generale in quanto riguarda Adobe App Builder. Ci sono molte buone risorse da trovare su Internet.
Per ulteriori informazioni sulla funzionalità di base di un proxy inverso, consulta le seguenti risorse esterne:

* [Cos&#39;è un proxy inverso](https://www.imperva.com/learn/performance/reverse-proxy/)
* [Cos&#39;è un proxy inverso e perché è importante](https://blog.hubspot.com/website/reverse-proxy)
